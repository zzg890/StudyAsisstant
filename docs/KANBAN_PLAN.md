# StudyAssistant 看板规划

## 1. 看板目标

用于跟踪 AI智学助手从 MVP 到增强版本的开发进度，覆盖：

- 前端页面与交互
- 后端 API 与数据层（EF Core + SQLite）
- AI 能力接入点
- 用户管理与登录能力
- AI 模型配置与运维能力
- 测试与发布准备

## 2. 看板列设计

建议使用 6 列：

1. Backlog（需求池）
2. Ready（已细化，可开发）
3. In Progress（开发中）
4. Review（代码评审/联调）
5. Test（测试验证）
6. Done（完成）

WIP 限制建议：

- In Progress: 最多 3 张卡
- Review: 最多 2 张卡
- Test: 最多 2 张卡

## 3. 优先级规则

- P0：阻塞主流程，必须优先完成
- P1：增强体验，MVP 后跟进
- P2：长期优化，按资源安排

排序规则：

1. P0 高风险/高依赖项优先
2. 先打通端到端闭环，再做功能丰富
3. 先可用，再优化体验与性能
4. 前端阶段优先完成应用壳层（layout + 导航栏）再推进业务页面细节
